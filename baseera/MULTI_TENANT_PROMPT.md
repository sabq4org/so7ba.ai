# 🔭 بصيرة — برومبت تصميم نظام متعدد العملاء (Multi-Tenant)

## الرؤية
تحويل بصيرة من أداة رصد داخلية إلى **منصة ذكاء إعلامي كخدمة (SaaS/B2G)** تخدم الوزارات والجهات الحكومية والشركات الكبرى.

كل عميل (وزارة/جهة) يحصل على **لوحة تحكم خاصة** ترصد وتحلل ما يهمه فقط.

---

## 🏗️ البنية المطلوبة

### 2. الكلمات المفتاحية (Keywords Engine)

#### أنواع الكلمات:
| النوع | الوصف | مثال |
|-------|-------|------|
| **أساسية (Primary)** | اسم الجهة + خدماتها الرئيسية | وزارة التجارة، التستر التجاري |
| **ثانوية (Secondary)** | مسؤولين + مواضيع مرتبطة | ماجد القصبي، السجل التجاري |
| **هاشتاقات** | هاشتاقات مستهدفة | #وزارة_التجارة |
| **حسابات** | حسابات X للمتابعة | @MCaborter |
| **منافسين** | جهات مشابهة للمقارنة | هيئة المنافسة |
| **مستبعدة** | كلمات تُفلتر من النتائج | تجارة عامة |

#### منطق البحث:
```
(أي كلمة أساسية OR أي هاشتاق) 
AND NOT (أي كلمة مستبعدة)
+ رصد الحسابات المستهدفة بشكل مستقل
```

### 3. لوحة التحكم لكل عميل

#### الصفحة الرئيسية (Dashboard):
- **حالة المشهد** — هادئ/محايد/متوتر/ساخن/أزمة (خاص بالعميل)
- **درجة أمان السمعة** — 0-100
- **ملخص ذكي** — AI يحلل ما يخص هذا العميل فقط
- **التوصية** — ماذا يفعل فريق الإعلام الآن
- **مؤشر الاتجاه** — تحسن/استقرار/تدهور

#### الرادار:
- أخبار تخص العميل فقط (مفلترة بالكلمات المفتاحية)
- تغريدات مرتبطة
- ذكر في وسائل إعلام

#### غرفة الأزمات:
- تنبيهات فورية عند ارتفاع الذكر السلبي
- تتبع الأزمة لحظياً
- اقتراح خطة استجابة AI

#### التقارير:
- يومي (8 صباحاً)
- أسبوعي (الأحد)
- لحظي عند الأزمات
- PDF قابل للتصدير

### 4. لوحة الإدارة (Super Admin)

صفحة خاصة بنا (بصيرة) لإدارة كل العملاء:

- إضافة/تعديل/إيقاف عميل
- إعداد الكلمات المفتاحية
- مراقبة استهلاك API
- تقارير الاستخدام
- إدارة الخطط والاشتراكات

### 5. نماذج جاهزة (Templates)

بدل ما نبدأ من صفر لكل عميل:

| القالب | الكلمات الافتراضية |
|--------|-------------------|
| 🏛️ وزارة | اسم الوزارة + الوزير + الخدمات الرئيسية + الهاشتاقات الرسمية |
| 🏢 هيئة حكومية | اسم الهيئة + الرئيس + القرارات + اللوائح |
| 🏦 شركة كبرى | اسم الشركة + CEO + المنتجات + المنافسين |
| 🏟️ حدث/فعالية | اسم الحدث + الهاشتاق + المنظم + التواريخ |
| 👤 شخصية عامة | الاسم + الألقاب + الجهة + المواضيع المرتبطة |

## 🔧 التصميم التقني

### الباكند (إضافات على الحالي):

```
/api/tenants/                    → CRUD العملاء
/api/tenants/{id}/keywords       → إدارة الكلمات المفتاحية
/api/tenants/{id}/dashboard      → لوحة التحكم الخاصة
/api/tenants/{id}/radar          → رادار مفلتر
/api/tenants/{id}/alerts         → التنبيهات
/api/tenants/{id}/reports        → التقارير
/api/tenants/{id}/crisis         → غرفة الأزمات
/api/admin/tenants               → إدارة شاملة (Super Admin)
/api/admin/usage                 → استهلاك API
```

### قاعدة البيانات (PostgreSQL):
```sql
-- العملاء
tenants (id, name, name_en, logo, plan, status, settings_json, created_at)

-- الكلمات المفتاحية
tenant_keywords (id, tenant_id, keyword, type, language, active)
-- type: primary, secondary, hashtag, account, competitor, excluded

-- الرصد والذكر
tenant_mentions (id, tenant_id, source, text, sentiment, emotion, url, author, created_at)
-- source: twitter, news, other

-- التنبيهات
tenant_alerts (id, tenant_id, type, severity, title, body, seen, created_at)

-- التقارير المولدة
tenant_reports (id, tenant_id, type, period, content_json, pdf_url, created_at)

-- مستخدمي كل عميل
tenant_users (id, tenant_id, email, password_hash, role, last_login)
-- role: admin, editor, viewer
```

### Scheduler معدّل:
```python
# بدل رصد عام واحد:
async def run_tenant_cycle():
    active_tenants = get_active_tenants()
    for tenant in active_tenants:
        # 1. جلب الكلمات المفتاحية
        keywords = get_tenant_keywords(tenant.id)
        
        # 2. بحث في تويتر (SocialData)
        tweets = search_twitter(keywords.primary + keywords.hashtags)
        tweets = filter_excluded(tweets, keywords.excluded)
        
        # 3. بحث في الأخبار (NewsData + GNews)  
        news = search_news(keywords.primary)
        
        # 4. تحليل المشاعر (Gemini)
        analysis = analyze_sentiment(tweets + news, tenant.name)
        
        # 5. حفظ النتائج
        save_mentions(tenant.id, tweets + news)
        update_dashboard(tenant.id, analysis)
        
        # 6. فحص الأزمات
        if analysis.crisis_score > tenant.alerts.crisis_threshold:
            send_alert(tenant, analysis)
```

### المصادقة (Auth):
- JWT tokens (access + refresh)
- كل عميل له مستخدمين (admin + viewers)
- Super Admin لنا فقط (بصيرة)
- middleware يتحقق من tenant_id في كل طلب

## ⚠️ ملاحظات تقنية

1. **SocialData** — الرصيد خلص، كل عميل إضافي = طلبات أكثر = تكلفة أعلى
2. **Gemini Flash** يظل الخيار الأمثل — رخيص وسريع مع عملاء متعددين
3. **الخصوصية حرجة** — كل عميل يشوف بياناته فقط، عزل تام بين العملاء
4. **Rate Limits** — SocialData + NewsData لهم حدود، لازم queue ذكي
5. **هذا برومبت تصميم فقط** — لا تنفيذ بدون إشارة خضراء
